## 🔒 开发规范和安全要求

### Git 提交规范

#### Commit Message 格式
```
<type>: <description>

[optional body]
```

#### Type 类型
- `feat`: 新功能
- `docs`: 文档更新
- `fix`: 修复问题
- `refactor`: 重构
- `chore`: 构建/工具相关

#### 示例
```bash
# 新增课程
feat: 创建课程 02-[课程名称]

# 更新文档
docs: 更新课程 02 需求文档

# 修复问题
fix: 修复课程 02 大纲中的时间错误
```

### 文件命名规范
- ✅ 使用中文文件名（带编号）
- ✅ 编号格式：`00-`、`01-`、`02-`
- ✅ 文件类型：`.md`
- ❌ 避免空格和特殊字符

### 文档结构规范
每个核心文档必须包含：
- ✅ 版本信息（版本、日期、说明）
- ✅ 清晰的章节结构
- ✅ 统一的标题层级（# ## ###）
- ✅ 适当的表格和列表
